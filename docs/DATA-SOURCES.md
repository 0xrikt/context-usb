# 数据源可行性调研报告

> 调研日期：2026-05-27
> 目标：评估 MVP 阶段各中国互联网平台的数据接入可行性

---

## 总览矩阵

| 平台 | 官方 API | 数据导出 | MCP Server | 接入难度 | MVP 优先级 |
|------|---------|---------|------------|---------|-----------|
| **Notion** | ✅ 完整 REST API | ✅ Markdown/CSV | ✅ 官方 MCP | 低 | P0 |
| **flomo** | ⚠️ 仅写入 API | ✅ HTML 导出 | ✅ 多个开源 MCP | 中 | P0 |
| **微信读书** | ❌ 无官方 API | ⚠️ 第三方工具 | ✅ 社区 MCP | 中 | P1 |
| **印象笔记** | ✅ 完整 API + SDK | ✅ ENEX 格式 | ❌ 无 | 中 | P1 |
| **微博** | ⚠️ 限制多 | ⚠️ 第三方工具 | ❌ 无 | 高 | P2 |
| **豆瓣** | ❌ 已关闭 | ❌ 无官方 | ❌ 无 | 高 | P2 |

## 逐平台详细分析

### 1. Notion（P0 - MVP 首发）

**可行性：高**

- **API**：官方 REST API，支持读取页面、数据库、块内容，创建/更新/删除操作
- **认证**：OAuth 2.0 或 Internal Integration Token（只读 scope 可选）
- **MCP**：官方 MCP Server 已发布（`@notionhq/notion-mcp-server`），支持 2026-03-11 API 版本
- **数据类型**：页面内容、数据库记录、评论、用户信息
- **限制**：Rate limit 3 requests/second，分页最大 100 条/页
- **接入方案**：直接使用官方 API + 官方 MCP Server

**对产品价值**：Notion 是中国知识工作者最常用的"第二大脑"，里面沉淀着用户的个人反思、年度复盘、目标规划等高质量个人上下文。

---

### 2. flomo（P0 - MVP 首发）

**可行性：中高**

- **API**：
  - **写入 API**：有，通过 Incoming Webhook URL 创建 memo（需 PRO 会员）
  - **读取 API**：**无官方读取 API**——这是关键限制
  - Rate limit：100 条/天（写入）
- **数据导出**：
  - HTML 格式全量导出（设置 → 账户详情 → 导出）
  - 每 7 天可导出一次
  - 支持按标签/搜索条件导出
- **MCP Server**：多个开源实现
  - `xianminx/mcp-server-flomo`（TypeScript）— 仅写入
  - `benbird-black/flomo-mcp`（Python，FastMCP）— 支持创建、查询、搜索、更新、删除
  - `chatmcp/mcp-server-flomo`（TypeScript）— 仅写入
- **接入方案**：
  - **方案 A（MVP 推荐）**：用户手动导出 HTML → 我们解析结构化。零 API 依赖，最稳定
  - **方案 B**：fork `benbird-black/flomo-mcp`，它声称支持读取（可能通过非官方接口）
  - **方案 C**：引导用户使用 flomo API 写入关键 memo 到我们的系统（反向集成）

**对产品价值**：flomo 是最典型的"个人树洞"——用户在这里记录碎片化的想法、情绪、灵感。这些数据是个人上下文的核心原料。flomo 创始人自己也提出了"个人上下文"概念，说明产品方向高度契合。

---

### 3. 微信读书（P1 - MVP 第二波）

**可行性：中**

- **API**：无官方公开 API，但有成熟的逆向方案
- **数据可获取**：
  - 书架列表
  - 划线/高亮内容
  - 个人想法/批注
  - 阅读时长统计
  - 热门划线
- **社区方案**：
  - Obsidian Weread Plugin（最成熟，导出到 Markdown）
  - 微信读书 MCP Server（社区开发，支持书架、笔记、划线管理，导出 Markdown/PDF）
  - 微信读书 Skills（2025 年新推出，通过 API Key 绑定账号）
- **接入方案**：
  - **方案 A（推荐）**：参考 Obsidian Weread Plugin 的逆向 API 接口，自建读取层
  - **方案 B**：使用社区 MCP Server
  - **风险**：逆向接口可能随微信读书更新失效，需持续维护

**对产品价值**：用户读什么书、划什么线、写什么想法，是理解一个人思想偏好和成长方向的极佳信号。

---

### 4. 印象笔记（P1 - MVP 第二波）

**可行性：中高**

- **API**：完整的 Evernote API，印象笔记版 host 在 `dev.yinxiang.com`
- **认证**：Developer Token（个人使用）或 OAuth（第三方应用）
- **SDK**：官方 Python SDK（`evernote-sdk-python`），Java、JS SDK 也有
- **数据可获取**：
  - 笔记本列表
  - 笔记内容（ENML 格式→需转换）
  - 标签系统
  - 搜索
  - 笔记元数据（创建/修改时间、地理位置等）
- **限制**：
  - ENML（Evernote Markup Language）格式需要特殊解析
  - Rate limiting 存在但文档不明确
- **接入方案**：使用官方 SDK + Developer Token，读取用户笔记并转换为 Markdown

**对产品价值**：印象笔记用户往往有多年沉淀的笔记，包括日记、工作反思、学习笔记等，数据量大且时间跨度长。

---

### 5. 微博（P2 - 后续迭代）

**可行性：中低**

- **API**：开放平台 `open.weibo.com`，OAuth 2.0 认证
- **可用端点**：
  - `users/show` — 用户资料
  - `statuses/home_timeline` — 首页时间线
  - `statuses/user_timeline` — 用户发布的微博
  - `statuses/show` — 单条微博详情
  - `comments` — 评论相关
  - `favorites` — 收藏
- **限制**：
  - 数据服务仅对企业开发者开放申请
  - 部分接口需高级权限（需申请）
  - 邮箱等敏感信息需单独授权
  - Rate limit 存在
  - **个人开发者获取完整权限困难**
- **替代方案**：
  - **stablog**（GitHub 开源）：专业微博备份工具，Windows + Mac
  - **微博备份助手**（Chrome 扩展）：导出 xlsx
  - **MemoMe/Speechless**：导出 PDF/TXT/Markdown
  - 油猴脚本方案：以页为单位备份，含图片
- **接入方案**：
  - **方案 A（推荐）**：用户使用第三方工具（stablog/Chrome 扩展）导出 → 我们解析
  - **方案 B**：以个人开发者身份申请 API，但权限可能不足
  - **方案 C**：申请企业开发者（需要公司资质）

**对产品价值**：微博是中国最大的公开"树洞"之一，用户发的微博、收藏、评论反映了兴趣、情绪、社交圈层。

---

### 6. 豆瓣（P2 - 后续迭代）

**可行性：低**

- **API**：**官方 API 已关闭**，不再接受个人应用申请
- **替代方案**：
  - 逆向工程（Fiddler/Charles 抓包豆瓣 App 接口）
  - 第三方爬虫服务（法律风险）
  - `douban_api`（GitHub 开源，基于爬虫恢复的 API 服务）
  - 豆瓣→Notion 同步工具（`Notion_sync_data`，同步书影音数据）
- **可获取数据**：
  - 标记的书/电影/音乐（想看/在看/看过 + 评分）
  - 短评/长评
  - 日记
  - 豆列
- **接入方案**：
  - **方案 A（推荐）**：通过已有的豆瓣→Notion 同步链路间接获取（用户先同步到 Notion，我们从 Notion 读）
  - **方案 B**：引导用户使用 `douban_api` 开源项目自建代理
  - **风险**：爬虫方案有法律风险（《反不正当竞争法》、用户协议），需评估

**对产品价值**：豆瓣标记是用户文化消费品位的最完整画像——读什么书、看什么电影、听什么音乐，这些是深层人格特征的外在表达。

## PIPL 合规要点

中国《个人信息保护法》（PIPL）对本产品的关键要求：

1. **明示同意**：处理敏感个人信息需取得用户单独同意（可能需书面同意）
2. **最小必要**：仅采集实现产品功能所必需的数据
3. **目的限定**：明确告知用户数据的使用目的和影响
4. **匿名化优先**：优先使用匿名化技术处理数据
5. **跨境传输**：如涉及跨境传输需额外合规（单独同意 + 安全评估/认证）
6. **2026 年新规**：修订后的《网络安全法》2026-01-01 生效，大幅提高罚则

**产品合规设计原则**：
- 用户主动授权导入，非被动采集
- 数据存储在用户本地或用户选择的云端（非我们的中心服务器）
- 提供完整的数据删除能力
- 不做跨境传输（服务器部署在中国大陆）
- 隐私协议清晰说明每类数据的用途

## MVP 数据接入路径总结

```
Phase 0（原型验证）：手动文件导入
├── flomo HTML 导出 → 解析
├── 微博导出文件（stablog/Chrome 扩展）→ 解析  
├── Notion 导出 Markdown → 解析
└── 用户粘贴文本 → 解析

Phase 1（MVP）：API 直连 + 文件导入
├── Notion API（官方，稳定）
├── flomo 文件导入（HTML 解析）
├── 印象笔记 API（官方 SDK）
└── 微信读书（社区逆向 API）

Phase 2（增长）：更多平台 + 自动同步
├── 微博 API（企业开发者）
├── 豆瓣（通过 Notion 中继）
├── 即刻 API
└── 定期自动刷新
```
