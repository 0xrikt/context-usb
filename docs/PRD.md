# Context USB — 产品需求文档 (PRD)

> 版本：0.1 MVP
> 日期：2026-05-27

---

## 一句话定义

**Context USB** 是中国互联网用户的个人上下文层——从你的"树洞"平台（flomo、Notion、微博、豆瓣等）提取你是谁、你在想什么、你经历了什么，让任何 AI 对话工具（豆包、DeepSeek、ChatGPT、Kimi）都能"插上 U 盘"般立即读懂你。

---

## 为什么做这个

### 用户痛点

1. **每换一个 AI 工具就要从零介绍自己**：用户在豆包聊了三个月个人成长，换到 DeepSeek 一切归零
2. **AI 不知道你的"内心世界"**：你读过什么书、写过什么日记、在微博上吐槽过什么——这些最能定义你的信息，AI 一无所知
3. **个人数据散落在十几个平台**：flomo 记想法、Notion 写复盘、微博发情绪、豆瓣标记读书——没有任何工具把这些串起来形成"你是谁"的完整画像
4. **聊个人成长时 AI 只会说正确的废话**：因为没有你的上下文，AI 只能给通用建议

### 市场空白（调研验证）

- **Unabyss**（最接近竞品）：做的是职业身份上下文（LinkedIn/Gmail/Slack），面向海外 builder。不覆盖中国平台，不面向个人成长场景
- **腾讯 IMA**：仅限微信生态，定位知识库不是个人上下文
- **flomo**：创始人提出"个人上下文"概念但只做记录工具，不输出给其他 AI
- **Mem0/Supermemory**：开发者基础设施，不是消费级产品
- **丘脑科技 Omni Memory**：SDK/API 层，无消费级产品

**结论**：无人做"中国树洞平台 → 结构化个人上下文 → 即插即用到任意 AI"的闭环。

---

## 目标用户

### 核心画像：「AI 原住民中的自我探索者」

- 25-35 岁，一二线城市
- 日常使用 2+ 个 AI 聊天工具（豆包、DeepSeek、ChatGPT、Kimi）
- 有个人成长意识：读书、写日记/memo、做年度复盘
- 在 flomo/Notion/微博/豆瓣等平台有 1 年以上的个人内容沉淀
- 痛感：跟 AI 聊深了发现它不懂我，每次都要重新解释背景

### 用户规模估算

- 中国 AI 聊天工具月活用户：豆包 ~8000 万 + DeepSeek ~数千万 + Kimi ~数百万
- 其中有"个人成长"聊天习惯的：估计 5-10%
- 同时在 flomo/Notion 等有内容沉淀的：估计占上述 20-30%
- **潜在用户**：百万级

## MVP 核心功能

### F1：数据导入 — 把你的"树洞"装进 U 盘

用户将个人数据导入 Context USB：

| 导入方式 | 支持平台 | MVP 阶段 |
|---------|---------|---------|
| **文件上传** | flomo（HTML 导出）、微博（stablog 导出）、任意文本文件 | Phase 0 |
| **API 直连** | Notion（OAuth）、印象笔记（Developer Token） | Phase 1 |
| **粘贴文本** | 任意来源的文本（日记、聊天记录、微信截图 OCR） | Phase 0 |

### F2：上下文结构化 — AI 把你的数据"读懂"

系统自动将导入的原始数据结构化为多层上下文文件：

```
context-usb/
├── identity.md        # 我是谁：性格特征、价值观、生活阶段
├── interests.md       # 我关注什么：阅读偏好、话题兴趣、审美取向
├── growth-journal.md  # 我在经历什么：近期思考、情绪变化、成长轨迹
├── relationships.md   # 我的社交世界：重要的人、社交模式
├── goals.md           # 我想去哪：短期目标、长期愿景、纠结的决定
└── voice.md           # 我怎么说话：表达风格、常用表达、幽默偏好
```

**结构化引擎**（LLM 驱动）：
1. 对导入的原始数据进行分类和提取
2. 生成自然语言描述（不是 JSON，是人类可读的 Markdown）
3. 标注每条信息的来源、时间、置信度
4. 用户可审阅和修改每一条

### F3：上下文输出 — 插到任何 AI 上

三种输出方式，覆盖不同技术能力的用户：

| 输出方式 | 用户操作 | 适合场景 |
|---------|---------|---------|
| **复制粘贴**（P0） | 一键复制上下文文本 → 粘贴到任意 AI 对话开头 | 所有用户，零门槛 |
| **MCP Server**（P1） | 配置 MCP 连接 → Claude/Cursor/Kimi Code 自动读取 | 技术用户 |
| **浏览器扩展**（P1） | 安装扩展 → 在豆包/DeepSeek/ChatGPT 网页版自动注入上下文 | 非技术用户，最佳体验 |

### F4：隐私控制 — 你决定 AI 看到什么

- **分级可见**：每个上下文文件可单独开关（如：给 AI 看兴趣，不看关系）
- **敏感信息标记**：系统自动标记可能敏感的内容（真名、地点、财务等），用户确认
- **本地优先**：数据默认存储在本地，云同步为可选项
- **一键清除**：随时删除所有数据

## MVP 不做的事

- ❌ 不做 AI 聊天界面（我们不是又一个 ChatGPT）
- ❌ 不做社交/分享功能
- ❌ 不做实时数据同步（MVP 阶段手动触发即可）
- ❌ 不接入需要爬虫的平台（豆瓣、小红书 → 后续迭代）
- ❌ 不做移动 App（先 Web + 浏览器扩展）

## 用户旅程

### 旅程 1：新用户首次使用（5 分钟内获得价值）

```
1. 打开 Context USB 网页
2. 注册/登录（邮箱或微信扫码）
3. 引导页：「选择你的树洞」
   ├── flomo → 引导导出 HTML 并上传
   ├── Notion → OAuth 授权
   ├── 印象笔记 → Developer Token 授权
   └── 直接粘贴 → 文本框
4. 等待 30-90 秒（loading 动画 + 进度提示）
5. 看到生成的上下文预览：
   「根据你在 flomo 的 342 条 memo，我了解到：
    - 你最近在思考职业转型方向...
    - 你对斯多葛哲学很感兴趣...
    - 你的表达风格偏直接、喜欢用类比...」
6. 用户审阅 → 修改/确认 → 保存
7. 一键复制上下文 → 粘贴到豆包/DeepSeek 开始对话
8. 💡 Aha moment：AI 第一次像"老朋友"一样跟你聊
```

### 旅程 2：日常使用——跟 AI 聊个人成长

```
1. 用户想跟 AI 聊"最近的焦虑"
2. 打开豆包/DeepSeek
3. 方式 A：手动粘贴上下文（30 秒）
   方式 B：浏览器扩展自动注入（0 秒）
4. 开始对话——AI 已经知道用户的背景、近期状态、思考风格
5. AI 的回应不再是通用建议，而是基于用户具体情况的个性化洞察
```

### 旅程 3：更新上下文

```
1. 用户在 flomo 写了新的 memo / 在 Notion 更新了复盘
2. 回到 Context USB
3. 点击"同步"→ 增量更新（只处理新内容）
4. 预览变化 → 确认
5. 下次跟 AI 对话时，上下文已包含最新内容
```

## 成功指标

### 北极星指标

**周活跃使用次数**（WAU × 每周使用上下文输出的次数）

### 关键指标

| 指标 | MVP 目标（上线 30 天） |
|------|---------------------|
| 注册用户 | 500 |
| 完成首次数据导入 | 60%（300） |
| 生成上下文并使用 ≥1 次 | 40%（200） |
| 7 日留存 | 25% |
| 上下文平均使用次数/用户/周 | 3+ |
